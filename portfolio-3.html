Projet CTF : Break the code

Lors de ma deuxième année de BUT, nous avions comme projet de créer un site pouvant héberger des “Capture the flag”. Les CTF sont des événements qui ont pour but de pratiquer la ludopédagogie en initiant les participants au piratage éthique au travers de défis de cybersécurité. Afin de créer ce site nous avons utilisé l’outil “CTFd” qui permet d’avoir tous les principaux éléments d’un CTF tout en étant modifiable car il est Opensource. Cela nous a permis de faire notre site plus facilement et plus rapidement, en se concentrant sur les challenges qui sont le cœur de ce projet.

Les challenges sont représentés sous forme de carte avec une description et un nombre de points à gagner. Pour réussir le challenge il faut entrer le “flag” qui est le code a entrer dans le challenge, que l’on obtient en venant à bout du défi.

Pour ma part, j’ai fait les défis Forensic qui consiste à fouiller dans une image disque que j’ai créé afin de trouver le flag. Pour ce faire j’ai utilisé une machine virtuelle Linux et j’ai créé une image disque avec des faux dossiers et fichier via le terminal linux.

J’ai aussi créé un défi web, le but étant de trouver le flag caché dans les cookies d’une page. Pour faire ce défi j’ai programmé en HTML pour la base du site, puis en PHP pour faire le cookie. Je me suis aidé d’un template pour le design du site.

Pour finir sur les challenges, j’ai créé un petit jeu vidéo sur godot en GDScript, qui consiste à un jeu de piste pour retrouver le mot de passe d’un ordinateur où se cache le flag. Je vous invite à regarder la section dédiée à ce jeu dans mon portfolio pour en savoir plus !

Au-delà des défis, je me suis occupée de la partie graphique du site. J’ai fait plusieurs thèmes de couleurs, ainsi que la page “Règles”. J’ai aidé au design des cartes des challenges. Tout cela a été codé en HTML et CSS.

Nous avons fait ce projet avec la méthode agile SCRUM. Nous avions des daily meeting et avions des rendez-vous avec le client toutes les deux semaines afin de discuter de l'avancée du projet. La gestion du projet s’est faite via GitLab, nous avons utilisé la partie Plan de gitlab en créant des issues et en organisant nos différents sprint sur l’issue board.
